Integration Guide

Connect AI agents to the Beefree SDK editor via MCP — two session models, one powerful result.

1. Editor-Managed Session

Available on paid plans · Simplest approach · New session per chat interaction

The editor owns the session. Call bee.startMcpSession() from the frontend — it returns a templateId that you pass to your backend agent. The agent uses it as the x-bee-template-id header when calling the MCP v2 endpoint. Each chat interaction creates a fresh session.

Frontend — start a session from the editor
// When user sends a chat message, start an MCP session:
const { templateId } = await bee.startMcpSession();

// Send templateId + user message to your backend agent
await fetch('/api/chat', {
  method: 'POST',
  body: JSON.stringify({ templateId, message: userMessage })
});
Backend — agent calls MCP v2
# Your agent sends MCP tool calls to:
# POST https://api.getbee.io/v2/sdk/mcp
# Headers:
#   Authorization: Bearer <CSAPI key>
#   x-bee-template-id: <templateId from startMcpSession()>
#
# The editor automatically reflects changes made by the agent.
# Each chat interaction = a new startMcpSession() call.

2. API-Managed Session + Co-editing

Higher plans with co-editing enabled · Persistent session · Editor + MCP stay aligned

Your backend creates the session via the API, then the editor joins it through co-editing. The connection is persistent — the AI agent and the visual editor operate on the same template simultaneously, with changes from both sides kept in sync automatically.

Step 1 — Create session server-side
import httpx

resp = httpx.post(
    "https://api.getbee.io/v2/sdk/mcp/template",
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={"template": existing_json}  # or {} for a blank template
)
template_id = resp.json()["templateId"]
Step 2 — Agent connects via MCP
# Point your MCP client at:
# POST https://api.getbee.io/v2/sdk/mcp
# Header: x-bee-template-id: {template_id}
#
# The agent can now call any MCP tool (beefree_add_section, etc.)
# All changes are applied to the shared template.
Step 3 — Editor joins via co-editing
// In the frontend, after editor is initialized:
bee.join(templateId);

// The editor now shows live updates from the AI agent.
// User edits in the editor are also visible to the agent.
Step 4 — Retrieve the final template
# When done, get the final result:
resp = httpx.get(
    f"https://api.getbee.io/v2/sdk/mcp/template/{template_id}",
    headers={"Authorization": f"Bearer {API_KEY}"}
)
final_template = resp.json()
